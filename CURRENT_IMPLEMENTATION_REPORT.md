## 4. PRODUCTION BLOCKERS

### CRITICAL
1. **Missing API routes** — frontend calls 4+ endpoints that return 404 (`/monitoring/drift/events`, `/monitoring/devices/{id}/trend`, `/compliance/frameworks/{fw}/summary`, `/compliance/rules`)
2. **No email delivery** — password reset and email verification tokens are never delivered to users
3. **IPWhitelistMiddleware blocks all non-localhost** — `SENSITIVE_PATHS` includes `/api/v1/users/` and `/api/v1/audit/` but whitelist only contains `127.0.0.1` and `::1`; in production this blocks all user management and audit APIs
4. **Demo passwords in README** — `admin123`, `engineer123`, `analyst123`, `viewer123` are hardcoded in README and `user_service.py`; `ensure_demo_users()` uses these weak passwords violating the 12-char policy
5. **Celery workers disabled** — all workers commented out in `docker-compose.yml`; bulk compliance, remediation, and scheduled polling will never execute

### HIGH
6. **`compliance_service.py` missing `created_at` field** — `get_all_latest_scores_by_tenant` uses `.distinct(ComplianceScore.device_id, ...)` but `ComplianceScore` has no `created_at` column (uses `generated_at`); query will error
7. **Migration enum mismatch** — migration 001 `userrole` enum has `admin, secops_lead, engineer, analyst, viewer`; models.py has `super_admin, admin, security_analyst, soc_analyst, auditor, secops_lead, engineer, analyst, viewer`; migration 002 adds the missing values via `ALTER TYPE` but only `super_admin`, `security_analyst`, `soc_analyst`, `auditor` — `DeviceType.GENERIC` is not in migration 001 enum either
8. **`conftest.py` uses wrong device_type** — `device_type="router"` is not a valid `DeviceType` enum value; tests will fail on insert
9. **`AuditLog` model has `password_hash` as `String(256)` but migration 001 uses `LargeBinary`** — schema mismatch

### MEDIUM
10. **No `commit()` in auth routes** — `register`, `verify_email`, `reset_password` call `db.flush()` but never `db.commit()` — data lost on disconnect
11. **Reports stored in Redis only** — `list_reports` derives reports from compliance scores; `get_report` requires Redis; if Redis flushes, all reports are lost
12. **`SIEMEvent` has no `created_at` field** — `siem.py` tries `event.created_at.isoformat()` on `ingested_at` — logic inconsistency
